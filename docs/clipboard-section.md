# Clipboard Section — Technical Description

## Overview

The Clipboard section is a quick notes/snippets manager for storing and managing text-based code snippets and notes. It provides a simple interface for creating, viewing, editing, and copying text snippets with optional language tags.

## Architecture

### Database Schema

**Table: `clipboard_items`**

| Column      | Type | Constraints    | Description                              |
|-------------|------|----------------|------------------------------------------|
| id          | TEXT | PRIMARY KEY    | UUID generated by crypto.randomUUID()    |
| content     | TEXT | NOT NULL       | The text content of the snippet          |
| language    | TEXT | NULL           | Optional language tag (e.g., "typescript", "python") |
| created_at  | TEXT | NOT NULL       | ISO 8601 timestamp of creation           |

**Indexes:** None (table is small, queries are simple)

### Data Model

```typescript
interface ClipboardItemJson {
  id: string;
  content: string;
  language: string | null;
  created_at: string;
}
```

## File Inventory

### Backend

- **`src/lib/clipboard-db.ts`** — Database layer
  - `initClipboardSchema(db)` — Creates clipboard_items table
  - `createClipboardItem(data)` — Insert new snippet
  - `getAllClipboardItems()` — Fetch all snippets, ordered by created_at DESC
  - `getClipboardItem(id)` — Fetch single snippet by ID
  - `updateClipboardItem(id, updates)` — Update content and/or language
  - `deleteClipboardItem(id)` — Delete snippet by ID

### API Routes

- **`src/app/api/clipboard/route.ts`**
  - `GET /api/clipboard` — Returns all clipboard items (newest first)
  - `POST /api/clipboard` — Creates new item
    - Body: `{ content: string, language?: string }`
    - Validates: content must be non-empty
    - Returns: 201 with created item

- **`src/app/api/clipboard/[id]/route.ts`**
  - `PUT /api/clipboard/[id]` — Updates existing item
    - Body: `{ content?: string, language?: string }`
    - Returns: 200 with updated item, or 404 if not found
  - `DELETE /api/clipboard/[id]` — Deletes item
    - Returns: 204 on success, 404 if not found

### Frontend

- **`src/app/clipboard/page.tsx`** — Main UI component
  - Create form (textarea + optional language input)
  - Card-based snippet list with infinite scroll
  - Inline editing mode for each snippet
  - Copy to clipboard functionality
  - Two-step inline delete confirmation

### Tests

- **`src/app/api/clipboard/route.test.ts`** — API route tests
  - GET returns array of items
  - POST creates item with content and language
  - POST returns 400 if content is missing

- **`src/app/api/clipboard/[id]/route.test.ts`** — Individual item route tests (if exists)

## Features

### 1. Create Snippet
- Textarea for content input (required)
- Text input for optional language tag
- "Add Snippet" button submits form
- Form clears after successful creation
- New snippet appears at top of list

### 2. View Snippets
- Card-based layout showing snippets
- Content displayed in monospace font with preserved whitespace
- Language tag shown as badge (if present)
- Created timestamp in local format
- Initial load shows 10 most recent snippets
- Infinite scroll loads 10 more when scrolling to bottom

### 3. Copy to Clipboard
- "Copy" button on each snippet
- Uses `navigator.clipboard.writeText()`
- Button shows "Copied!" feedback for 2 seconds
- No syntax highlighting (plain text copy)

### 4. Edit Snippet
- "Edit" button switches card to edit mode
- Inline textarea and language input
- "Save" commits changes via PUT request
- "Cancel" discards changes and returns to view mode
- Only one snippet can be edited at a time

### 5. Delete Snippet
- First click enters a temporary confirm state for that card
- Second click within 3 seconds performs the deletion
- No browser confirmation dialog is used
- List refreshes after deletion

### 6. Infinite Scroll
- Uses IntersectionObserver API
- Triggers when scroll reaches bottom sentinel element
- Loads 10 more items each time
- Stops when all items are displayed

## Data Flow

### Create Flow
```
User fills form → Submit → POST /api/clipboard
  → createClipboardItem() → INSERT INTO clipboard_items
  → Return new item → Refresh list → Display at top
```

### Update Flow
```
User clicks Edit → Inline form appears → User edits → Save
  → PUT /api/clipboard/[id] → updateClipboardItem()
  → UPDATE clipboard_items → Return updated item
  → Exit edit mode → Refresh list
```

### Delete Flow
```
User clicks Delete → card enters confirm state → second click confirms
  → DELETE /api/clipboard/[id] → deleteClipboardItem()
  → DELETE FROM clipboard_items → 204 response
  → Refresh list
```

### Copy Flow
```
User clicks Copy → navigator.clipboard.writeText(content)
  → Show "Copied!" feedback → Reset after 2s
```

## Common Pitfalls

1. **Empty content validation**: The API validates that content is non-empty and trimmed. The UI also requires the textarea to be filled.

2. **Language field is optional**: The database allows NULL for language. The UI treats empty string as undefined when submitting.

3. **Infinite scroll observer cleanup**: The IntersectionObserver must be disconnected in the useEffect cleanup to prevent memory leaks.

4. **Edit mode state**: Only one snippet can be in edit mode at a time. Starting edit on a new snippet replaces the previous edit state.

5. **Copy behavior**: The page prefers `navigator.clipboard.writeText()` and falls back to a hidden textarea + `document.execCommand("copy")` path for older environments.

6. **Delete confirmation timeout**: The confirm-delete state auto-resets after 3 seconds and is cleared on unmount.

7. **Whitespace preservation**: Content is displayed in a `<pre>` tag with `whitespace-pre-wrap` to preserve formatting while allowing wrapping.

## Future Enhancements (Not Implemented)

- **Syntax highlighting**: The `language` field is stored but not used for highlighting yet. Future implementation could use a library like Prism.js or Highlight.js.

- **Search/filter**: No search functionality currently. Could add text search across content and language tags.

- **Tags/categories**: Currently only supports a single language field. Could expand to support multiple tags or categories.

- **Export/import**: No bulk export or import functionality.

- **Snippet templates**: No template system for common snippet patterns.
